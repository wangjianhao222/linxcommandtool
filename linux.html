Linux System Information Tool

A Comprehensive System Monitoring Utility for Linux

Overview

The Linux System Information Tool (linux.py) is a Python-based utility designed to collect and display detailed system information for Linux environments. By leveraging Python's subprocess module and standard Linux commands, it provides a consolidated view of system metrics, including OS details, CPU, memory, disk usage, network interfaces, running processes, logged-in users, and system logs. This tool is ideal for system administrators, developers, and Linux enthusiasts seeking an efficient way to monitor and diagnose system performance.

Features

System Information

  • Retrieves OS metadata (name, ID, version, build) from /etc/os-release.
  • Displays hostname, kernel version, architecture, uptime, and load averages using commands like uname and uptime.

CPU Information

  • Extracts CPU model, physical CPU count, core count, and logical processors from /proc/cpuinfo.

Memory Information

  • Reports total, free, used, buffered, and cached memory, plus swap space details, from /proc/meminfo.
  • Converts memory values to GB for readability.

Disk Information

  • Shows disk usage statistics with human-readable output using df -h.

Network Information

  • Collects interface details, including IPv4/IPv6 addresses, traffic statistics, and error counts using ip -s a.
  • Converts byte counts to MB for clarity.

Process Information

  • Lists the top 20 processes by CPU and memory usage with ps aux --sort=-%cpu,-%mem.

Logged-in Users

  • Displays active user sessions using who.

System Logs

  • Retrieves the latest 15 log entries, preferring journalctl with a fallback to dmesg.

Requirements

  • Python: Version 3.6 or higher.
  • Operating System: Linux (e.g., Ubuntu, CentOS, Debian).
  • Dependencies: Standard Python libraries (subprocess, os, platform, re, time).
  • System Commands: Requires cat, uname, df, ip, ps, who, journalctl, and dmesg in the system PATH.
  • Permissions: Elevated privileges may be needed for some commands (e.g., dmesg).

Installation

Prerequisites

Ensure Python 3.x is installed:

python3 --version

Install Python if needed:

sudo apt install python3  # Debian/Ubuntu
sudo yum install python3  # CentOS/RHEL

Verify system commands:

which cat uname df ip ps who journalctl dmesg

Setup

  1. Download the script:
    2. wget <script_url>/linux.py
  2. Set executable permissions:
    3. chmod +x linux.py
  3. Run the script:
    4. python3 linux.py

Usage

Run the script to display system information in the terminal:

python3 linux.py

The output is organized into sections: System, CPU, Memory, Disk, Network, Processes, Users, and Logs.

Advanced Usage

  • Selective Execution: Modify the main() function to call specific functions (e.g., get_cpu_info()).
  • Automation: Schedule with cron: 
    0 * * * * python3 /path/to/linux.py >> /path/to/system_report.log
  • Custom Output: Adjust process count in get_process_info() or log lines in get_system_logs().

Troubleshooting

  • Command Not Found: Install missing tools (e.g., apt install iproute2 procps).
  • Permission Issues: Run with sudo for restricted commands.
  • Incomplete Output: Ensure system services are running and commands are supported.
